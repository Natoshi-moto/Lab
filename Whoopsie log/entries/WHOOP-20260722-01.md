## What happened (plain English)

Six of seven hard gates say they can turn green when an “ODS-SEC” case passes. Those cases live as **spec-only** text. The real diagnostic suite file has **zero** of them.

So the only practical way a gate goes green is a **waiver**, not a test. Worse: the docs say “expected FAIL,” which is easy to hear as “known, so fine.”

## What we thought was true at the time

That writing the case IDs was almost as good as having the tests — a roadmap counted as progress.

## What was actually true

A red you decided to expect is still red. A green path that cannot run is a whoopsie dressed as governance.

## How it was noticed (or not)

Fable’s BREAK test rigor review (PR #70) mapped gates → ODS-SEC → code and found the empty set.

## What we did about it (if anything)

Recorded in Fable response + VETO checklist. **Not fixed** on main at time of this entry.

## What we still might be wrong about

How many other “GREEN only if …” conditions in the Lab also point at vapor.
