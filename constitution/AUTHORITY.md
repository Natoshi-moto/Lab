# Authority constitution

## Principle

Nexus separates **purpose**, **proposal**, **authorization**, **execution**, **observation**, and **promotion**. No participant acquires a later power merely by performing an earlier function.

## Principals

| Principal | May | May not |
|---|---|---|
| Human operator | define purpose, approve mutations, merge, tag, release, stop work | convert unsupported prose into evidence by declaration |
| AI research seat | inspect routed context, reason, propose files or observations | self-authorize execution, self-promote findings, silently widen scope |
| Router | select declared context, compile route packs, record exclusions | determine truth, grant undeclared capabilities |
| Executor | run an allowlisted operation against an exact baseline | infer new purpose or alter policy |
| GitHub | preserve commits, proposals, reviews, checks, tags and releases | confer epistemic truth by merge or status badge |
| Nexus verifier | check syntax, hashes, manifests, scopes and local invariants | certify semantic correctness outside checked properties |

## Mutation authority

- `main` is accepted working state, not certified truth.
- A branch or pull request is a proposal space.
- A frozen tag or snapshot is immutable `CANONICAL_AS_IS` history.
- Audit output has `status_authority: NONE` and lives outside its target.
- No AI account may grant itself broader read, write, execution, or promotion authority.

## Enforcement boundary

The local tools enforce route-path safety, deterministic snapshot verification, observation schema rules, target binding, hash-chain integrity, and selected privacy checks. GitHub branch rules and repository permissions remain remote administrative controls and must be configured separately. Policy text is not represented as stronger enforcement than the software actually provides.
