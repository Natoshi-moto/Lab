# Decision

The Nexus Research Lab is one private Git repository containing the durable corpus, governance, system source, operations, evidence, failures, audit overlays and snapshots.

“One repository” means one authoritative namespace and recoverable history. Large, secret or local-only payloads may use declared storage classes, but their manifests and identities remain governed by this repository.

Physical paths express stable lifecycle and authority. Domain and programme maps refer to an object by identity rather than copying it into multiple subject folders.

## Consequences

- The web app, terminal, API, GitHub interface and model routes must converge on repository-addressable objects.
- Derived indexes and caches are rebuildable projections, not a second authority store.
- AI accounts are attributed seats, not owners of project memory or policy.
- GitHub is amplified by Nexus semantics rather than treated as a formatting constraint.
