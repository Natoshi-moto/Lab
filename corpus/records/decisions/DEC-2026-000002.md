## Resolution

The operator's explicit, human-authorized decision is to **keep the repository public** rather than restore private visibility. This record and its accompanying commit revise the documents and code that assumed private visibility to match:

- `NEXUS.json`: `visibility_requirement` changed `PRIVATE` → `PUBLIC`; `visibility_policy_status` changed from `PRIVATE_REQUIREMENT_CONFLICT_HUMAN_DECISION_REQUIRED` to `RESOLVED_PUBLIC_PER_DEC_2026_000002`.
- `constitution/PRIVACY.md`: repository-perimeter statement corrected to state the repository is public; the `LAB_PRIVATE` data-class routing rule corrected to no longer claim repository visibility as its enforcement boundary, with an explicit note that a full classification redesign remains separately open (not decided by this record).
- `registry/providers/providers.json`: the `github` provider's `data_boundary` changed `PRIVATE_REPOSITORY` → `PUBLIC_REPOSITORY`.
- `system/nexus_lab/github.py`: the bootstrap visibility gate (`require_private_visibility`, refusing anything but `PRIVATE`) inverted to `require_public_visibility`, refusing anything but `PUBLIC`; repository creation now passes `--public` instead of `--private`; receipt claims text updated; receipt schema bumped `v2` → `v3` since its claims now assert the opposite of what a `v2` receipt asserted. `tests/test_github.py` updated to match.
- `docs/GITHUB_BOOTSTRAP.md`, `docs/GITHUB_ADMIN_CHECKLIST.md`: rewritten to describe the public-requiring flow; a `History` note added to `GITHUB_BOOTSTRAP.md` recording that it required `PRIVATE` from R001 until this reversal.
- `snapshots/restore/README.md`: dropped the stale "private" qualifier.
