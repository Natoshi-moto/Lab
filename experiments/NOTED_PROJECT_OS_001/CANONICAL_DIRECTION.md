- “We shipped the **host + membrane + receipted contribution loop** so a sim *can* be designed, run, verified, and shared *as part of the project*.”

- Embedding / pinning a sim-frame is a **later stage**, not the definition of success for the current one.

## 3. IN / FOR / BUILT INTO

### IN — where design and project work live

- Noted: projects, notes, canvas, prompts, App Design  
- Lab: experiments, packages, REGISTRY, Checkpoint doctrine  
- Multi-seat AI: Codex implement / Claude debug / Grok drive with routes and receipts  

Design of scenarios, invariants, non-claims, and attack plans lives **in** this system — not only in disposable chat.

### FOR — what the system is optimized to serve

- Deterministic runs others can re-run  
- Shareable identity of a run (e.g. seed + config/package pin + receipt roots)  
- Contribution / reputation / synthetic standing bound to **verified** results  
- Cold drop of a workbench, not an archaeology dump  
- Hard firewall: never endorse real-world economic value on project objects  

Host, bridges, optional Nostr mesh, creature standing surfaces, and Lab ops exist **for** that shared-result economy — not for generic notes-only SaaS and not for a tradable token.

### BUILT INTO — structural hooks before full transplant

The architecture must leave a clear organ slot even before the organ is fully present:

- Registered package path for sim-frame (e.g. under `products/`, REGISTRY-bound)  
- Bridge/capability hooks: run, verify, import run-pack (approval-gated where effectful)  
- Noted-facing records: scenario, run pack, verify result (as first-class project objects over time)  
- Checkpoint tags and seed project graph  
- Drop layout that can include sim-frame when ready  
- Seat packets that sequence: **system first, instrument second**

## 4. What the “simulator economy” means here

Not: live staking, redeem rails, or real-world market claims.

Yes:

```text
Participation  = run pinned scenarios, produce receipts
Shared truth   = same pin + params + seed → same verifiable results
Contribution   ≈ verified runs, breaks, repairs against rules
Reputation     ≈ others reproduced your results
Tip / standing = synthetic appreciation (non-redeemable)
Authority      = who may admit scenarios into the pack (Lab / REGISTRY / operator)
```

The scarce good is **credible, reproducible experimental credit** inside a synthetic membrane — not a bank balance.

Historical nexsim work (deterministic receipts, strict manifests, domain-separated draws, long-run baselines) is the **lineage of this job**. It is not automatically the vendored product of this phase. Lab-native pin or reimplementation under containment remains the admission path.
