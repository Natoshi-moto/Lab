# Plain-language summary (for YouTube / Reddit viewers)

status_authority: `NONE`. This is one directed AI review, not an independent audit.

**What this is.** The operator wrote a set of official-looking release documents for the Nexus / Noted project — a README, a system overview, a white paper, and a technical spec — and asked another AI to attack them for honesty. I'm that reviewer. My job was not to decide whether the project is good, but whether these documents can be published as-is.

**The good news first.** These documents are honest to an unusual degree. Instead of hiding the project's rough edges, they list them out loud: an old "wallet" screen that still talks about balances and sending, a Nostr social feature that stores a secret key in the browser the wrong way, a creature-game module whose packaged file is missing, and a "simulation engine" that lives on the operator's computer but isn't actually part of the project yet. I checked each of these against the real code, and the documents were telling the truth. The core software pieces that *are* claimed to work — exporting your notes as a graph, running the local tests, building the app — really do work when I ran them.

**The catch, and why I say "not ready to call a release."** The documents' headline promise is "STRICT NO SALE" — the idea that the points, tips, and creatures in the app are just data, like WoW gold, and can never be bought, sold, or cashed out, *and that the software itself will enforce this.* Right now that promise is written down but not built. There's no code that blocks the forbidden money-words, no test that rejects a sale, and the old wallet screen with "Send" and "Stake" buttons is still clickable. The project also promises to list every place the app phones out to the internet, but that list isn't in the release yet (the app quietly contacts font servers, a code CDN, and AI providers). So my verdict is **INCOMPLETE**: publish these papers *only* as what they honestly call themselves — an early draft waiting for outside review — and never as a finished, validated, or secure release. The single biggest open question is whether "no sale" can ever really be enforced in software that anyone is free to copy and change. Until someone builds and independently tests that enforcement, it's a good intention, not a guarantee.
