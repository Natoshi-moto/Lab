## 4. Piggyback strategy (defensible, buildable)

### Year-0 (isomorphic, cheap)

- Keep **GitHub** as remote, PRs, Actions, permissions, public repo glass.  
- Semantic objects live **in-repo** (this pack, LOOM captures, SEC-ID registries, receipts).  
- “Mithub” v0 = **docs + conventions + bot/check** that fails CI if:
  - claim cited without record path  
  - SEC-ID referenced in prose but missing in code/registry  
  - epoch-0 NEX UI without RESET banner (when that code exists)  
- Optional: GitHub App / Action named Mithub that comments CLAIM-IDs on PRs.

### Year-1 (portal)

- Web UI that reads GitHub API + in-repo semantic index.  
- Agent job runner (Hermes/pipeline) authenticated as a seat, write-only via PR.  
- Search across **code + claims + transcripts** (public-safe).  
- Course and break/status boards as first-class views.

### Year-N (optional deeper substrate)

- Only if needed: content-addressed semantic object store **mirroring** git, not replacing it without a migration constitution.  
- Still piggyback identity (GitHub auth, or later portable identity) where sane.

**Do not** start by rewriting git. That fails isomorphism with “we use GitHub to record everything.”
