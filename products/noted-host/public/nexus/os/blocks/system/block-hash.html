BLOCK HASH verify what you're about to install

What this is for. Before you install or run a block someone sent you, hash it here and compare against the hash they advertised. Different bytes → different hash → different block, regardless of filename. Same hash on two copies means the bytes are byte-identical. See docs/COMMUNITY_BLOCKS_DESIGN.md for the full trust story.

Declared manifest (extracted from the block)

What the block claims it will do. The kernel will only let it do this much. These channels come from the block's DECLARE message — we extract them by scanning the source. Trust this only after the hash matches a hash you trust. A block can lie about its manifest; the protocol catches the lie at runtime, but by then you've already opened it.
