# AI-Assisted Development Demonstration v0.04

## What this archive demonstrates

This archive is both an app and a method for building apps with AI. It demonstrates:

1. A runnable product shell.
2. A nested router architecture for block applications.
3. A specification carried inside the source archive.
4. A planning packet that lets a future AI worker continue without conversation memory.
5. Load-bearing code excerpts that bind theory to current files.
6. Type and schema contracts that can become implementation surfaces.
7. A verification gate that treats the zip as the unit of trust.

## Demonstration path for a future reviewer

1. Extract the archive.
2. Read `HANDOFF.md`.
3. Read `PROJECT_NOTES.md`, `CONTEXT.md`, and `BUILD_BLOCKS.md`.
4. Inspect `specs/NOTED_NEXUS_AI_ASSISTED_TECHNICAL_SPEC_v0.04.md`.
5. Run `npm ci` if dependencies are absent.
6. Run `npm run typecheck`.
7. Run `npm run build`.
8. Launch the app.
9. Confirm `/` lands on `/nexus-router`.
10. Confirm Nexus OS opens inside the Noted shell.
11. Confirm Prompt Studio and Nexus Agent remain available through Nexus.
12. Confirm the next build block is BB-01, not a random feature burst.

## Why this matters

AI code generation is easy. AI continuity is hard. This archive focuses on continuity.

The planning packet says what the project is. The context says what must not be violated. The build blocks say what to do next. The handoff says how a new worker re-enters. The technical spec gives the architecture. The schemas and registries give machines something finite to validate. The zip binds all of it to code.
